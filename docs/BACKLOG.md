## 완료 (2026-06-02 — 전사 실시간 진행률 게이지, 사용자 요청) · 커밋 7509cbd
- [x] **실시간 진행률 게이지** — WhisperKit `segmentDiscoveryCallback`(발견 세그먼트 최대 end / 오디오 길이)로 **레이스 없이** 실제 % 산출. 모달 JobRow 결정형 게이지+%, **모달 닫아도 우측하단 플로팅 미니 게이지**(원형, 탭하면 모달 재오픈). 멀티트랙 스케일, 단조 증가 가드, 0.99 캡(화자 구분 단계 정체 회피).

> 검증: 빌드 그린 · 단위 37개. 적대적 리뷰(16 에이전트, 9 확정) 반영 — 핵심: `nonisolated(unsafe)` progress 폴링이 16-워커 동시 변경과 **데이터 레이스(UB)** → segmentDiscoveryCallback으로 교체. 진행률 역행·100% 정체도 수정.

## 완료 (2026-06-02 — P0 취소 + 상시 추가버튼 + 이름변경) · 커밋 2a82210
- [x] **전사 작업 취소(개별/모두)** ★사용자요청 — `cancelJob`/`cancelAllJobs`. WhisperKit이 `Task.checkCancellation()`을 전사 경로에서 호출함을 소스로 확인 → **워커 Task 취소로 in-flight 전사를 즉시 중단**(긴 파일이 워커를 점유해도 탈출 가능). 취소된 Task 재사용 오염은 `while !Task.isCancelled` 드레인 가드 + 신규 워커 재시작으로 차단(직렬성 불변식 유지). `discardIfCancelled`로 취소를 '실패'가 아닌 '폐기'로 전환.
- [x] **취소 버튼 UI** — `JobRow` 진행/대기 작업 개별 취소(✕), 진행 모달 footer '모두 취소'(활성 2개 이상).
- [x] **상시 '오디오 추가' 버튼** ★사용자요청 — 툴바 `+` → 다중 파일 importer → `enqueueFiles`(녹음이 있어도 항상 추가).
- [x] **녹음 이름 변경** ★사용자요청 — `renameRecording`(빈/공백/동일값 가드+persist) + 사이드바 우클릭 '이름 변경…' 시트.
- [x] **적대적 리뷰(워크플로우, 27 에이전트) 반영** — ① 실패 작업이 모달 자동닫힘에 영구히 숨겨지던 버그(닫힘 기준 `activeJobCount`→`jobs.isEmpty`) ② diarize 취소 비인지 → 취소 시 diarize 시작 생략 ③ RenameSheet onSubmit 빈 이름 가드.

> 검증: 빌드 그린 · 단위 28개 통과(취소 4종 + 기존 직렬성 유지). 통합 스위트(large-v3/diarize)는 fixture 있을 때만, 이번엔 스킵.

## 완료 (2026-06-02 구현)
- [x] **전사 결과 안 보임 버그 (근본 원인: 동시 전사 ANE 경합)**
  - 증상: 여러 파일을 연속 드롭하면 일부가 전사 안 됨(segments=0 저장). 같은 파일이 한 번은 60, 한 번은 0.
  - 근본 원인: 각 드롭이 별개 Task로 같은 WhisperKit 인스턴스를 동시 호출 →

  - 해결: AppState를 **단일 직렬 워커 큐**로 개편(동시 전사 0). 빈 결과는 저장 안 하고 실패 표시.
- [x] **드래그앤드롭 전사 큐잉** — 전사 중에 또 드롭하면 큐 뒤에 추가, 순차 처리.

- [x] **완료된 것부터 즉시 목록에 등장** — 선택하면 바로 자막(전사) 표시.
